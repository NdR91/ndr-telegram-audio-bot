Provider

💡 Provider ≠ Modello

  • Un provider è una connessione API (OpenAI, Gemini, OpenRouter…).
  • Un modello è un'AI specifica offerta da un provider (es. gpt-4o, whisper-1).
  • Stesso provider, modelli diversi = capacità diverse.
  • Esempio: con OpenRouter, whisper-1 per trascrizione e gpt-4o per refinement.
  • I fallback permettono di specificare modelli alternativi in caso di errore del primario.

Modalità pipeline

Scegli il tipo di elaborazione e poi assegna i modelli alle fasi.

🔀

Due fasi: trascrizione + refinement

L'audio viene prima trascritto da un modello STT, poi raffinato da un modello testo. Consente di usare modelli specializzati per ogni fase.

  • ✅ Modello STT dedicato (es. Whisper)
  • ✅ Modello testo separato per refinement (es. GPT, Claude)
  • ⚠️ Costo doppio (due chiamate API)
  • ⚠️ Latenza maggiore
🔄

Singolo passaggio

Un unico modello multimodale gestisce trascrizione e refinement in una chiamata. Ideale per modelli come Gemini che supportano input audio diretto.

  • ✅ Una sola chiamata API
  • ✅ Latenza ridotta
  • ⚠️ Richiede modello con capacità single_pass_audio_to_text
  • ⚠️ Non tutti i provider supportano questa modalità
🌟
